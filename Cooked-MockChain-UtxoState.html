newtype UtxoState Source #

A UtxoState provides us with the mental picture of the state of the UTxO graph: Each address has a set of UTxOs that consist in a value and some potential datum.

newtype UtxoValueSet Source #

Represents a set of values, yet, we use a list instead of a set because Value doesn't implement Ord and because it is possible that we want to distinguish between utxo states that have additional utxos, even if these could have been merged together.

utxoStateDiffTotal :: UtxoStateDiff -> Value Source #

Computes the total difference in value between a new state and an old state: utxoStateDiff new old. If this difference is positive for some token t, it means some t it was burnt. If it is negative, some t was minted.

utxoStateDiff :: UtxoState -> UtxoState -> UtxoStateDiff Source #

Computes the difference between two states.

utxoStateDiffSrc :: UtxoStateDiff -> UtxoState Source #

Returns the source of a difference. Satisfies: utxoStateDiffSrc (utxoStateDiff a b) == a

utxoStateDiffTgt :: UtxoStateDiff -> UtxoState Source #

Returns the target of a difference. Satisfies: utxoStateDiffTgt (utxoStateDiff a b) == b

Pretty-printing

prettyUtxoState :: UtxoState -> Doc ann Source #

Pretty prints a UtxoState. The entire point of producing a UtxoState instead of a UtxoIndex is to provide the user a picture that is closer to their mental model of what is going on. Hence, we must be able to display our UtxoStates in a human readable fashion.
